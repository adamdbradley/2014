Purpose of a Promise
Helps us deal with concurrency by
expressing code in more brain friendly
 flow-control style but in a way that 
we can also rely on to behave 
in a trustworthy manner
Promises have state
Fulfilled: 
    The action relating to the promise succeeded
Rejected: 
    The action relating to the promise failed
Pending:
    Hasn't fulfilled or rejected yet
Settled: 
    Has fulfilled or rejected 
The State is Trustworthy

The Old Style
function loadImg(imgsrc, callback) {
  var i = new Image();
  i.onload = callback(i);
  i.src = imgsrc;
}

loadImg(url, function(img) {
  console.log("Img Loaded!");
  // take img and do something *
});
The Promise Style
function loadImg(imgsrc) {
  return new Promise(function(resolve, reject) {
    var i = new Image();
    i.onload = resolve(i);
    i.src = imgsrc;
  }
}

loadImg(url).then(function() {
  console.log("Img Loaded!")
  // take img and do something
});
